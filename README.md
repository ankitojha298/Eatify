# LocalBite - Production Ready MERN App

LocalBite is a comprehensive restaurant discovery and food ordering platform built with the MERN stack (MongoDB, Express, React, Node.js). 

## Features
- **Authentication**: JWT based Login/Signup with Roles (Admin/User).
- **Core App**: Browse restaurants, view menus, add to cart, mock Stripe checkout.
- **Admin**: Dashboard with revenue tracking, order stats, and restaurant count.
- **Modern UI**: Built with React, Vite, Tailwind CSS, and Lucide icons.

## Step-by-Step Setup Instructions

### 1. Prerequisites
- Node.js installed (v16+)
- MongoDB connection string (Atlas or Local)
- Stripe Account (for Test API keys)

### 2. Backend Setup
1. Open a terminal and navigate to the `backend` folder:
   ```bash
   cd backend
   ```
2. Install dependencies (already done if generated by assistant):
   ```bash
   npm install
   ```
3. Create a `.env` file in the `backend` directory with the following variables:
   ```env
   NODE_ENV=development
   PORT=5000
   MONGO_URI=your_mongodb_connection_string
   JWT_SECRET=your_jwt_secret_key
   STRIPE_SECRET_KEY=your_stripe_test_secret_key
   ```
4. Start the backend development server:
   ```bash
   npm start
   # or `npm run dev` if nodemon is configured
   ```

### 3. Frontend Setup
1. Open a new terminal and navigate to the `frontend` folder:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Start the Vite development server:
   ```bash
   npm run dev
   ```

### 4. Application Usage
- Open `http://localhost:3000` (or whatever port Vite gives) in your browser.
- Register an account.
- Note: To see the Admin Dashboard, modify your user's role to `admin` directly in your MongoDB database, or modify the fallback in `backend/controllers/userController.js` to assign admin on first signup.

## Deployment Guide

### Deploying the Backend on Render
1. Push your code to a GitHub repository.
2. Go to [Render.com](https://render.com) and create a new **Web Service**.
3. Connect your GitHub repository and select the `backend` folder as the Root Directory.
4. Set the Build Command to: `npm install`
5. Set the Start Command to: `node server.js`
6. Add all the Environment Variables from your `.env` file into Render's dashboard.
7. Click **Deploy**. Note the deployed URL (e.g., `https://localbite-api.onrender.com`).

### Deploying the Frontend on Vercel
1. Go to [Vercel.com](https://vercel.com) and create a new Project.
2. Import your GitHub repository.
3. Edit the **Framework Preset** to Vite.
4. Set the **Root Directory** to `frontend`.
5. Before deploying, you MUST update the API URL in `frontend/src/utils/api.js`. Change the `baseURL` to your deployed backend URL.
    ```javascript
    const api = axios.create({
      baseURL: 'https://localbite-api.onrender.com/api',
      // ...
    });
    ```
6. Click **Deploy**. Vercel will automatically build and host your frontend.

### Final Checks
- Ensure CORS on the backend allows requests from your Vercel frontend URL. 
- You can use the `cors` package in `backend/server.js`:
  ```javascript
  app.use(cors({ origin: 'https://your-vercel-app.vercel.app' }));
  ```
